# デザインシステム手引き（ポートフォリオ用）

元記事: 「AIでおしゃれな画面を作るためのデザインシステムを学ぼう！」(Qiita, yusuke_ando_vj)
このサイト（HCI/HBI/デザイン/モデリング訴求の3D作品空間, `/redesign/`）向けに絞って再構成したもの。

---

## 読んだ感想

元記事は**LP・サービスサイト向け**の網羅的チェックリスト。14項目どれも正しいが、そのまま適用すると「情報を過不足なく伝える無難なサイト」に寄る。今回のサイトの勝ち筋は**逆**で、情報量を削って"操作させて能力を見せる"一発勝負。なので元記事の枠組みは目次として借りつつ、中身はほぼ書き換えている。

刺さった概念は3つだけ:
- **Design Token化**（色・余白・角丸を変数で持つ）→ そのまま採用。今後の調整が全部1箇所で効くようにする。
- **UI状態の明示**（Hover/Active/Disabled等）→ 3Dオブジェクトの `hover / drag / focus` にそのまま翻訳できる、良い借用元だった。
- **「曖昧な指示を避ける」の章** → 自分たちのやりとり（"アーティストらしさ"→"HCI/デザイン訴求"の往復）そのもの。指示は毎回このレベルまで具体化する。

逆に、このサイトには要らないと判断したもの:
- ページ構成の定石（Hero→Problem→Solution→…の縦読み導線）— 1画面完結の空間型UIなので直訳できない
- カラー役割の細分化（Success/Warning/Error等）— フォーム的UIが無いので過剰
- レスポンシブの厳密な分岐 — 本領はPC操作前提と明言済みなので、スマホは簡易表示で割り切ってよい

## 1. 目的・ターゲット

- **目的**: 採用担当・デザイナーに「このサイトの作者は自分で触れるUIを設計・実装できる」と一目で伝える
- **ターゲット**: HCI/HBI・デザイン・モデリング職の採用担当者。デザインの見た目より「動かしたときの手応え」で評価する層
- **避けるべき失敗**: 研究者としての自己紹介に寄ること（材料科学の文脈は背景に留める。[[hci-portfolio-direction]] 参照）

## 2. 情報設計・ページ構成

LPの縦読み導線（課題提起→解決→実績→CTA）は不採用。かわりに：

1. **最初の3秒で「これは触れる」と伝える**（プローブ/掴む、の一文提示）
2. **本体は1画面完結の空間**（結晶格子 or 3Dオブジェクト空間）
3. **個々の作品情報はホバー/クリックで後出し**（最初から全部見せない）
4. 連絡導線は最後に一箇所（footer相当）だけ、控えめに

## 3. レイアウト

- カラム/グリッドの概念は使わない。**空間配置**（フィボナッチ球面 / Voronoi粒界）が代わり
- 情報密度は意図的に低く保つ。「読ませる」でなく「触らせる」

## 4. ビジュアルスタイル

現行案Dの位置づけ: **ダークUI × 3D UI**（元記事の分類でいうと最も近い）
- ダークUI: 背景 `#0d0e12` 基調、開発者/AI/テック系の訴求と相性が良い
- 3D UI: 立体オブジェクト直接操作。読み込み負荷は許容（ポートフォリオは重さより体験優先）
- 避けた組み合わせ: グラスモーフィズム単体（近未来感は出るが操作力の証明にならない）、ミニマル単体（洗練はするが無難に戻る）

## 5. トーン&マナー

`テック的 × 遊び心 × 手応えがある` の3語。
避けたい印象: 「きれいなポートフォリオテンプレ」「研究者の自画像」「BMOに寄りすぎた可愛さ一辺倒」

## 6. カラー設計（Design Token）

現行 `redesign/index.html` の `:root` 変数がそのまま Design Token。今後もここだけ触れば全体に効く。

```css
--font:'Space Grotesk',system-ui,sans-serif;   /* 差し替え候補: GoogleSansFlex.ttf など */
--mono:'Space Mono',ui-monospace,monospace;
--bg:#0d0e12;      /* 背景 */
--ink:#f2f3f7;     /* 本文文字 */
--sub:#8b90a2;     /* 補足文字 */
--accent:#ff6a3d;  /* CTA・強調（暖色1色に絞る） */
```

作品カテゴリカラー（分類の視覚言語、そのまま維持）:

```
hw(ハードウェア)    #ff6a3d
sw(ソフトウェア)    #46d4c8
robo(ロボティクス)  #c792ea
design(デザイン)    #ffce5c
prod(プロダクト)    #a0e060
```

## 7. タイポグラフィー

- 見出し・UI文言: **Space Grotesk**（デザイン系で映えるグロテスク、既定）
- コード・ラベル・座標表記: **Space Mono**（"設計者っぽさ"を出す小道具として活用）
- 差し替え可否: `--font` 1行で全体に反映。リポジトリ内 `GoogleSansFlex.ttf` への切替も可
- 日本語本文が増える場合のみ、元記事の「行間1.7〜1.8」目安を適用（今は英語主体UIなので未適用）

## 8. コンポーネント設計 → 3Dオブジェクトへの翻訳

元記事のUI部品発想を、このサイトの語彙に翻訳したもの:

| 元記事の部品 | このサイトでの相当物 |
|---|---|
| Card | 3Dオブジェクト1個（作品1件） |
| Button (Primary) | オブジェクトをクリック→フォーカス |
| Modal | 右側 info パネル（フォーカス時に表示） |
| Tooltip | ホバー時の `#tip`（作品名だけ先出し） |
| Accordion | ダブルクリックでフォーカス解除→格納 |

## 9. UI状態（3Dオブジェクトの状態設計）

元記事のHover/Active/Disabled等の概念を直訳:

```
Default : ホームポジションで緩く浮遊、emissive弱
Hover   : emissive強め、tooltip表示、cursor:grab
Drag    : cursor:grabbing、慣性速度を記録、controls無効化
Release : 速度→回転として保持、緩やかにホームへ回帰
Focus   : 拡大(scale 1.18)、右パネル表示、autoRotate停止
```

Disabled/Error/Loading相当は無し（フォーム的操作がないため不要 — 元記事のUI状態を鵜呑みにせず要不要を判断した箇所）

## 10. 余白・角丸・影

物理的な"間"に相当するのは **慣性の減衰率とホームへの復元力**。
現行値: `vel *= 0.93`（毎フレーム減衰）、`pull = (home - pos) * 0.012`（復元力）。
- 値を上げる → キビキビして機械的な手応え
- 値を下げる → ふわっと有機的な手応え
どちらに振るかは今後の調整ポイント（本人フィードバック待ち）

## 11. レスポンシブ設計

**PC操作を前提として明言済み**。スマホ/タッチはタップ＝フォーカスのみ対応し、掴む/放るの慣性演出は無理に作り込まない（元記事の「レスポンシブは必須」を鵜呑みにしない判断）。

## 12. アクセシビリティ

- 起動失敗時にエラーを画面に表示する仕組み（build D-2で導入済み）＝コントラスト云々より前の"動くことの保証"を優先
- カテゴリ色は色だけに依存せず、記号（`Kb` `Bm` 等の2文字シンボル）を必ず併記 → 色弱対応にもなる副次効果

## 13. デザインシステム化・Design Token

- 実装済み: CSS変数によるToken化（フォント・色・サブカラーを`:root`に集約）
- 未実装（今後のTODO）: 慣性パラメータ（減衰率・復元力）も変数化し、サイト上部の隠しパネル等で調整可能にすると本番でも微調整しやすくなる
